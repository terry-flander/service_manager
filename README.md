# The Flying Bike — ServiceDesk

Field service management for The Flying Bike mobile bicycle repair business. Handles job scheduling, customer management, invoicing, email integration, and reporting.

![Python](https://img.shields.io/badge/Python-3.11+-blue) ![Flask](https://img.shields.io/badge/Flask-3.1-green) ![SQLite](https://img.shields.io/badge/SQLite-3-lightgrey) ![Docker](https://img.shields.io/badge/Docker-Compose-blue)

---

## Features

- **Job management** — Three job types (Booking, Workshop, Rental) with separate fields and reference sequences (FB-, PB-, RB-)
- **Customer database** — Full customer records with merge, import, and live search
- **Scheduling calendar** — FullCalendar view with job events, region dates, and custom events. Drag to reschedule.
- **Invoicing** — HTML and PDF invoices with GST-inclusive, GST-exclusive, and GST-exempt tax modes. Shop tickets for workshop jobs.
- **Email integration** — Gmail polling via OAuth2 turns booking emails into jobs automatically. Replies sent from within the app with template substitution and optional invoice attachment.
- **Sales reporting** — Date-range report by job type and status. GST breakdown. CSV export.
- **Region management** — Service regions with bookable dates, suburb mapping, and calendar integration
- **Admin tools** — User management, 2FA (TOTP), DB backup, job and customer CSV import, audit log

## Quick Start (local development)

### 1. Clone and set up environment

```bash
git clone <repo>
cd field_service
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env` — at minimum set `SECRET_KEY`:

```bash
# Generate a secure key
python3 -c "import secrets; print(secrets.token_hex(32))"
```

### 3. Run

```bash
python app.py
# → http://localhost:5000
# Default login: admin@flyingbike.com.au / changeme123
```

The database is created automatically on first run. Seed data (parts, regions, suburbs) is loaded from CSV files.

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `SECRET_KEY` | **Yes** | Flask session secret — long random string |
| `GOOGLE_MAPS_API_KEY` | No | Enables address autocomplete on job forms |
| `GMAIL_USER` | No | Gmail address for email polling |
| `GMAIL_CLIENT_ID` | No | OAuth2 client ID (Google Cloud Console) |
| `GMAIL_CLIENT_SECRET` | No | OAuth2 client secret |
| `GMAIL_REFRESH_TOKEN` | No | Generated by `gmail_oauth_setup.py` |
| `GMAIL_LABEL` | No | Gmail label to poll (default: `Booking Email/Open Bookings`) |
| `GMAIL_POLL_MINUTES` | No | Poll interval in minutes (default: `5`) |
| `DATA_DIR` | No | DB directory (default: project root; Docker uses `/data`) |

### Setting up Gmail integration

```bash
# One-time OAuth2 setup — opens browser to authorise
python3 gmail_oauth_setup.py
# Copy the GMAIL_REFRESH_TOKEN it prints into .env
```

## Docker Deployment

### Prerequisites

- Docker + Docker Compose on the server
- `.env` file with production values
- SSL certificates in `certs/` (or use the AWS ALB setup in `DEPLOY.md`)

### Run

```bash
docker compose up -d

# Run DB migrations after any schema update
docker compose exec -w /app flask python3 migrate.py

# View logs
docker compose logs -f flask

# DB backup — downloads via browser (admin login required)
# GET /admin/backup-db
```

The database persists in the Docker volume `servicedesk_app_data` at `/data/field_service.db`.

For full AWS EC2 + ALB setup (HTTPS, health checks, deployment runbook) see **`DEPLOY.md`**.

## Job Types

| Type | Reference | Created from | Key fields |
|---|---|---|---|
| **Booking** | `FB-0001` | Website form / manual | Region, suburb, date, start/end time, service types, bike description |
| **Workshop** | `PB-0001` | Walk-in | Date only (defaults to today), bike description |
| **Rental** | `RB-0001` | Manual | Start date, end date, address |

### Tax Treatment

| Setting | Invoice behaviour |
|---|---|
| Tax Inclusive | Prices include GST. Back-calculates: GST = total ÷ 11 |
| Tax Exclusive | Prices ex-GST. Adds 10%: total = subtotal × 1.1 |
| GST Exempt | No GST. Displays "nil" on invoice. |
| Cash payment | Always GST-free regardless of tax setting |

## Seed Data Notes

- **`parts.csv`** is loaded on **every startup** and upserts all parts — manual DB edits to parts will be overwritten on next restart. Edit the CSV to make permanent changes.
- **`regions_suburbs.csv`** is loaded **once only** (skipped if any regions already exist). To reload, clear the `regions` and `suburbs` tables first.
- Default admin account (`admin@flyingbike.com.au` / `changeme123`) is created only if it doesn't already exist. **Change the password immediately in production.**

## Development Notes

- No test suite. No linter configured.
- `sqlite3.Row` objects throughout — access columns as `row['name']`, not `row.name`.
- `PRAGMA foreign_keys = ON` is set on every DB connection.
- Per-user preferences (job filters, report selections, calendar view) are stored in the `settings` table — key pattern: `job_filter_{user_id}`, `report_prefs_{user_id}`, etc.
- Status colours are stored in `settings` as `status_color_<status>` with hardcoded fallback defaults.
- `jobs.tax_inclusive` is an `INTEGER` (0, 1, or 2) — never cast to Python `bool` before passing to `calc_totals()`.
- See **`CLAUDE.md`** for detailed architecture notes intended for AI-assisted development.
